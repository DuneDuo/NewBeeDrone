## 一、核心架构决策

### 1.1 树莓派 = 纯转发，不过脑子

```
Pi 只做两件事：
├── 进程1：MAVLink 转发 — FC UART ↔ Server TCP
└── 进程2：ArUco 解算   — OpenCV → LANDING_TARGET → FC TELEM2
```

| 应该放 Pi | 应该放服务器 |
|-----------|-------------|
| 字节透传 | dispatch 消息路由 |
| 断线重连 | 状态收集 (status_collector) |
| 啥都别管 | ACK 追踪 (command_tracker) |
| | 心跳监控 (heartbeat_watchdog) |
| | 日志 (logger) |
| | Web 面板 |

**不在 Pi 上做安全决策**——断连保护全部走飞控 failsafe（FS_GCS_ENABLE、FS_THR_ENABLE、RTL）。

### 1.2 视频推流

- 图传成品输出 RTSP
- Pi 上 `ffmpeg` 转推 RTMP 到服务器 SRS，转 HLS 给浏览器播放
- 不用 frp——Pi 主动往外连，NAT 方向顺的，不需要打洞

## 二、pymavlink 底层机制

### 2.1 `wait_heartbeat()` — 有坑，别用

```python
# ❌ 不要用
conn.wait_heartbeat()
conn.target_system  # 可能返回 0

# ✅ 手动收
m = conn.recv_msg()
if m.get_type() == 'HEARTBEAT':
    sys_id = m.get_srcSystem()
```

**原因**：`recv_msg()` 内部调 `probably_vehicle_heartbeat()` 过滤非飞行器心跳：

```python
# pymavlink/mavutil.py line 362-370
def probably_vehicle_heartbeat(self, msg):
    if msg.type == MAV_TYPE_GCS:       # ← GCS 类型被过滤
        return False
    if msg.autopilot == MAV_AUTOPILOT_INVALID:  # ← 也被过滤
        return False
    return True
```

mock_server 发的是 `MAV_TYPE_GCS` + `MAV_AUTOPILOT_INVALID`，全中过滤条件，`target_system` 保持 0。

### 2.2 `get_msgbuf()` vs `pack()` — 转发必用前者

```python
# ❌ pack() 会覆盖 sysid/compid（用当前连接的身份）
server.write(msg.pack(server.mav))  # → sys=255(树莓派)

# ✅ get_msgbuf() 保留原始身份
server.write(msg.get_msgbuf())      # → sys=1(飞控)
```

`get_msgbuf()` 返回接收时的**原始字节**，不重新编码。

### 2.3 `parse_char()` vs `parse_buffer()` — 用后者

```python
# ❌ 新版 pymavlink parse_char() 要求 Sequence[int]，逐字节喂会炸
mav.parse_char(byte)  # TypeError: can't extend bytearray with int

# ✅ parse_buffer() 一次喂整坨字节，内部自动缓冲拼接
for msg in mav.parse_buffer(data):
    print(msg.get_type())
```

### 2.4 TCP — pymavlink 只支持客户端

`mavtcp` 类里只有 `connect()`，没有 `bind()` + `listen()` + `accept()`。服务器端得原生 socket + pymavlink 编解码：

```python
# 服务器端骨架
server_sock = socket.socket(...)
server_sock.bind(("0.0.0.0", 57600))
server_sock.listen(1)
client_sock, addr = server_sock.accept()

mav = mavlink.MAVLink(None, srcSystem=254, srcComponent=190)

# 发：手动 pack + send
msg = mav.heartbeat_encode(...)
client_sock.send(msg.pack(mav))

# 收：recv + parse_buffer
data = client_sock.recv(4096)
for msg in mav.parse_buffer(data):
    ...
```

## 三、连接方式对比

| | UDP | TCP |
|---|---|---|
| 可靠性 | 丢包不重传 | 可靠交付 |
| 连接模型 | 无连接（sendto/recvfrom） | 有连接（connect/accept） |
| 端口冲突 | 两个进程不能绑同一端口 | 一个监听，多个客户端连 |
| pymavlink 支持 | 双向（udp:）/ 只发（udpout:）/ 只收（udpin:） | 只客户端 |
| 适用场景 | SITL 仿真、本地调试 | 树莓派→服务器生产链路 |

### UDP 端口绑定规则

```
udp:host:port     → 绑端口 + 收发（input=True 默认）
udpout:host:port  → 不绑端口，只发
udpin:host:port   → 绑端口，只收
```

## 四、转发程序架构

### 4.1 最终代码结构

```
Pi/
├── config.py      # 连接参数
├── transport.py   # open_connection(fc/server) + 手动心跳检测
├── main.py        # 死循环重连 + 双向 get_msgbuf() 转发
└── mock_server.py # 模拟 GCS（测试用）
```

### 4.2 main.py 核心设计

```
while True:                    ← 外层：永远不退出
    fc/conn = None
    try:
        fc = open_connection("fc")
        server = open_connection("server")
        while True:            ← 内层：收发循环
            msg = fc.recv_msg()     → server.write(msg.get_msgbuf())
            msg = server.recv_msg() → fc.write(msg.get_msgbuf())
    except:
        close(fc) + close(server)   ← 双重 try 兜底 close
        sleep(3)
```

五个可能的断点：`recv_msg()` 两端各一个、`write()` 两端各一个、autoreconnect 失败。全被外层 try 兜住。

## 七、MAVLink 飞行指令测试

### 7.1 测试流程

```
GUIDED 模式 → Arm → Takeoff 10m → SET_POSITION_TARGET_GLOBAL_INT(南50m) → Land
```

全部指令从服务器端 mock_server.py 发出，经 Pi 转发到 SITL，COMMAND_ACK 原路返回。

### 7.2 关键指令

| 指令 | msg ID | 说明 |
|------|--------|------|
| `MAV_CMD_DO_SET_MODE` | 176 | param1=1(CUSTOM_MODE), param2=4(GUIDED) |
| `MAV_CMD_COMPONENT_ARM_DISARM` | 400 | param1=1(arm), param2=0(尊重安全检查) |
| `MAV_CMD_NAV_TAKEOFF` | 22 | param7=高度(m) |
| `SET_POSITION_TARGET_GLOBAL_INT` | 86 | 直接设位置，lat/lon * 1e7 |
| `MAV_CMD_NAV_LAND` | 21 | 降落 |

### 7.3 位置计算

- 纬度 1 度 ≈ 111km，1 米 ≈ 0.000009 度
- 南 50m = lat - 0.00045
- API 传 `lat * 1e7`（整数）

## 八、常见坑

| 坑 | 现象 | 原因 | 解决 |
|----|------|------|------|
| wait_heartbeat 返回 0,0 | 身份校验失败 | probably_vehicle_heartbeat 过滤 GCS | 手动 recv_msg() |
| UDP 端口冲突 | 两个进程抢端口 | 都绑同端口 | 一方用 udpout: |
| TCP 监听 | ValueError: host:port required | pymavlink 只支持客户端 | 原生 socket + pymavlink 编解码 |
| parse_char 报错 | can't extend bytearray with int | 新版要求 Sequence[int] | parse_buffer(data) |
| close() 崩溃 | NoneType has no 'close' | autoreconnect 把 port 设 None | try/except 兜底 |
| SITL sys=1,comp=1 不是 1,0 | 身份不匹配 | ArduCopter autopilot 组件是 1 | 改 config |
| `-35.363...` 坐标 | SITL 家在哪 | ArduPilot 默认 Canberra 测试场 | 正常，不影响测试 |
