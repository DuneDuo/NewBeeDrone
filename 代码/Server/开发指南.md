## 一、Server 上跑了什么

一个 `python run.py` 进程，三层在里面：

```
run.py
  ├── daemon 线程: Drone.serve()        mavlink 层  ✅ 快写完
  ├── daemon 线程: mission_loop()       mission 层  🟡 你写
  └── 主线程: Flask app.run()           web 层      🟡 你写
```

**依赖方向：** `web → mission → mavlink`。上层 import 下层。下层不知道上层存在。

---

## 二、三层怎么通信（核心思路）

**不靠回调、不靠事件。Drone 实例本身就是共享数据。**

```
飞控发来一条 MAVLink 消息
  │
  ├─→ dispatch(drone, msg) → handler 把 msg 字段写成 drone 的属性
  │   例: drone.lat = msg.lat / 1e7
  │       drone.last_ack_result = msg.result
  │
  └─→ mission 层每秒轮询 Drone.registry
       读 drone.last_ack_result → ACK 成功? → 推进状态机
       调 drone.send_command_long() 发下一条指令
```

- **mavlink 层** 不知道 mission 存在。handlers 只写 drone 属性。
- **mission 层** 不知道 MAVLink 怎么收的。只从 registry 读属性，决定下一步发什么。
- **web 层** 不知道飞控协议。只读 registry + mission 状态，通过 HTTP/WebSocket 暴露给前端。

## 四、mavlink 层暴露的接口（你写 mission 时直接调）

### 4.1 查飞机

```python
from mavlink.drone import Drone

Drone.registry              # dict[int, Drone]   例: {1: <Drone>, 2: <Drone>}
drone = Drone.registry.get(sys_id)   # 不在线返回 None
```

飞机连上来收到第一个 HEARTBEAT 就注册进 registry，断开自动移除。

### 4.2 发指令

```python
drone.send_command_long(command_id, param1=0, ..., param7=0)
drone.param_request_read(b"参数名")       # 读飞控参数
drone.param_set(b"参数名", 值, param_type=9)  # 写飞控参数
```

指令 ID 和参数含义查 **`学习资料/mavlink协议解读/ArduPilot指令集/中文解释/01-通用指令集-common.md`**，搜 `MAV_CMD` 开头的枚举。

### 4.3 drone 实例属性（handlers 写的）

| 属性 | 来源消息 | 含义 |
|------|------|------|
| `drone.lat / lon / alt` | GLOBAL_POSITION_INT | GPS 坐标 + 相对高度 |
| `drone.last_ack_command` | COMMAND_ACK | 上一条指令的 ID |
| `drone.last_ack_result` | COMMAND_ACK | 0=成功, 1=暂拒, 2=拒绝, 3=不支持, 4=失败 |
| `drone.flight_mode` | HEARTBEAT | 当前飞行模式编号 |
| `drone.armed` | HEARTBEAT | 是否已解锁 |

你的第一步就是在 `mavlink/handlers.py` 里把这些 `@register` 补上，然后在 `drone.py` 的 `__init__` 里给这些属性设默认值 `None`。

## 五、mission 层 — 你要写的核心

### 5.1 你需要学的东西

| 技能 | 用途 |
|------|------|
| Python 状态机模式 | 管理任务从 IDLE→ARM→TAKEOFF→... 的状态转换 |
| MAVLink COMMAND_LONG 指令集 | 知道每一步发什么指令、参数什么意思 |
| `Drone.registry` 的用法 | 查飞机在线、读位置、读 ACK |
| Python `threading` | `mission_loop` 作为 daemon 线程 1Hz 轮询 |

### 5.2 你的代码放哪

```
Server/
  mission/
    __init__.py
    executor.py       ← MissionExecutor 类 + 状态机逻辑
    tasks.py          ← 任务类型定义
```

### 5.3 工作机制

mission 层不靠 mavlink 层主动通知。一个 daemon 线程每秒轮询：

```python
def mission_loop():
    while True:
        for drone_id in list(Drone.registry.keys()):
            executor.tick(drone_id)
        time.sleep(1)
```

`tick(drone_id)` 每次做的事：
1. `drone = Drone.registry.get(drone_id)`
2. 读 `drone.last_ack_result` — 有没有新 ACK
3. 读 `drone.lat/lon/alt` — 当前位置
4. 根据"当前状态 + ACK + 位置"决定下一步
5. 调 `drone.send_command_long()` 发指令
6. 把已处理的 `drone.last_ack_result` 置 `None`（消费掉）

### 5.4 状态机结构

```
IDLE → ARMING → TAKEOFF → EN_ROUTE → LANDING → DONE
  ↑                                                │
  └──────────── FAILED ←────────────────────────────┘  (任何步骤失败 → RTL)
```

每个状态的转换由一个 ACK 成功（`last_ack_result == 0`）触发。web 层调 `executor.start_delivery()` 从 IDLE 跳到 ARMING。

### 5.5 关键注意点

- **ACK 消费**：读完立刻置 None，否则下个 tick 周期重复推进
- **指令超时**：发指令时记 `time.time()`，5 秒没收到 ACK → 重试或失败
- **位置到达判断**：用 Haversine 球面距离公式，别直接减经纬度。阈值设 2~3 米

## 六、web 层 — Flask + WebSocket

### 6.1 你需要学的东西

| 技能 | 用途 |
|------|------|
| Flask 路由 + JSON | REST API，`request.json`、`jsonify` |
| WebSocket | 服务器主动推数据给浏览器 |
| 前端 HTML + JS | 一个简单的监控面板 |

### 6.2 要写的路由

| 路由 | 方法 | 功能 | 数据来源 |
|------|------|------|------|
| `/api/drones` | GET | 在线飞机列表 | `Drone.registry` |
| `/api/drone/<id>` | GET | 单架飞机详情 | `Drone.registry[id]` |
| `/api/mission/start` | POST | 启动任务 | 调 `executor.start_delivery()` |
| `/api/mission/status/<id>` | GET | 任务状态 | `executor.current` |
| `/ws` | WebSocket | 每秒推状态 | registry + executor |

### 6.3 前端页面

最简单的监控面板：飞机列表 + 地图标点（Leaflet.js）+ 启动任务按钮 + 紧急 RTL 按钮。
