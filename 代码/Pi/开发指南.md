## 一、Pi 上跑了什么

树莓派只有一个 Python 进程，`main.py` 是入口。目前进程里只有一个模块：

```
main.py
  └── relay 模块  ✅ 已写完
       飞控 ←→ 服务器 的 MAVLink 双向转发
```

你的任务是在这个进程里加一个 **cv 模块**：

```
main.py
  ├── relay 模块  飞控 ←→ 服务器 MAVLink 转发  ✅
  └── cv 模块    ArUco 视觉定位 + 修正降落      🟡 你写
```

两个模块在同一个进程里，共享飞控的 MAVLink 连接。relay 建连，cv 拿过来发位置修正指令。

## 三、relay 模块的关键接口

### 3.1 `open_connection(target)` — 建连

```python
from transport import open_connection

fc     = open_connection("fc")      # → pymavlink 连接对象, 失败返回 None
server = open_connection("server")  # 同上
```

函数做了四件事：读 config → `mavutil.mavlink_connection()` 建连 → 等第一个 HEARTBEAT → 比对 sys_id/comp_id 确认身份 → 返回。你的 cv 模块只关心 `fc`。

### 3.2 pymavlink 连接对象的三个操作

```python
msg = fc.recv_msg()          # 阻塞收消息，返回 MAVLink 消息对象或 None
fc.write(msg.get_msgbuf())   # 发原始字节
fc.mav.send(msg)             # 用 mav 对象发（和上面等价）
fc.target_system             # 飞控的系统 ID（建连时自动填的）
```

### 3.3 编码一条 MAVLink 指令

语法模式：`fc.mav.消息类型_encode(参数...)` 返回消息对象 → `fc.mav.send(msg)` 发出。

cv 模块需要用到两类指令的位置修正参数，详见参考文档。

完整 API 参考：**`学习资料/mavlink协议解读/pymavlink-API完整参考.md`**（第 4 节"发送消息"）

完整指令参数：**`学习资料/mavlink协议解读/ArduPilot指令集/中文解释/01-通用指令集-common.md`**（搜 `SET_POSITION_TARGET_GLOBAL_INT` 和 `MAV_CMD_NAV_LAND`）

## 四、cv 模块要做什么

```
CSI 摄像头 → OpenCV 读帧 → 检测地面 ArUco 码 → 计算偏移量 → 发位置修正指令给飞控
```

### 4.1 你需要学的东西

| 技能 | 用途 |
|------|------|
| OpenCV 基础 | 读摄像头帧、灰度转换 |
| ArUco 检测 + 位姿估计 | `detectMarkers()` + `estimatePoseSingleMarkers()` |
| 相机标定 | 获取内参矩阵和畸变系数（做一次就固定） |
| 坐标系转换 | OpenCV 坐标系 → 飞控 NED 坐标系 |
| pymavlink 发指令 | `fc.mav.set_position_target_global_int_encode()` |
| Python 多线程 | `threading.Thread`，cv 不阻塞 relay |

### 4.2 你的代码放哪

```
Pi/
  cv/
    __init__.py
    detector.py      ← ArUco 检测类
    controller.py    ← 降落修正逻辑（读 tvec → 算偏移 → 发指令）
```

然后在 `main.py` 里 `import` 你的模块，起一个 daemon 线程跑降落修正。

### 4.3 关键注意点

- **坐标系转换**：OpenCV（x右 y下 z前）≠ 飞控 NED（x北 y东 z下）。根据摄像头安装方向做映射。最简单的方式：放一个码在地上，飞机悬停上方，看 tvec 实测值手算关系。
- **阈值**：偏移小于 ~10cm 别发修正，避免抖动。
- **帧率**：不要在 while True 里全速跑 OpenCV，加 `time.sleep(0.03)` 控制。

## 六、调试方法（按顺序）

**1. 链路测试（不涉及你的代码）**

电脑跑 `python mock_server.py`，Pi 的 config 指向电脑 IP，Pi 跑 `python main.py`。看日志确认双向转发正常。

**2. ArUco 单独测试**

电脑上 USB 摄像头写独立脚本，只做 ArUco 检测 + 画框 + 打印 tvec。确认检测到码、坐标合理。调通再搬 Pi。

**3. SITL 仿真测试**

启动 ArduCopter SITL，Pi 连 SITL 的 UDP 端口，发位置修正指令，看仿真飞机是否响应。

**4. 合入 main.py，端到端测试。**
