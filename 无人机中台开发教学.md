## 1. 前置知识：网络通信基础

### 1.1 TCP 是什么

TCP 像一个**电话通话**：

| 打电话的动作 | TCP 对应的函数 |
|-------------|---------------|
| 买电话机 | `socket.socket()` |
| 插电话线，设分机号 | `bind(('0.0.0.0', 端口号))` |
| 等人打进来 | `listen()` + `accept()` |
| 拨别人的号码 | `connect((IP地址, 端口号))` |
| 说话 | `send(字节数据)` |
| 听对方说话 | `recv(缓冲区大小)` |
| 挂电话 | `close()` |

**TCP 保证三件事：**
- 数据一定送到
- 顺序不乱
- 内容不错

### 1.2 IP + 端口 = 电话号码

```
IP 地址     = 区号          （比如 8.152.207.82）
端口号      = 分机号        （比如 5760）
socket     = 那一部电话机

完整地址：  8.152.207.82:5760  =  "打到 8.152.207.82 这个地址的第 5760 号分机"
```

端口号范围 0-65535，但 0-1023 是系统保留的（如 80 是 HTTP，22 是 SSH）。你自己的程序**从 1024-65535 里随便选**，别跟其他程序冲突就行。

### 1.3 服务端 vs 客户端

```
服务端（你的服务器）               客户端（飞机树莓派）
     │                                    │
     │  1. socket()   买电话              │  1. socket()   买电话
     │  2. bind()     插线，公布号码       │
     │  3. listen()   开机等电话           │
     │  4. accept()   接电话  ←────────────│  2. connect()  拨号
     │                连接建立！           │
     │  5. recv()     听对面说话 ←────────│  3. send()     说话
     │  6. send()     回复      ────────→ │  4. recv()     听回复
     │  7. close()    挂电话              │  5. close()    挂电话
```

**关键点：** 必须先启动服务端，客户端才能连上。就像你必须先开机等人打电话，别人才能打进来。

### 1.4 动手验证 1：两个终端互发消息

开两个终端，都在同一台电脑上跑。

**终端 1（服务端，先跑）：**
```python
import socket

server = socket.socket()                    # 买电话
server.bind(('0.0.0.0', 12345))            # 插线，分机号 12345
server.listen(1)                            # 等人打进来
print('等待连接...')

client, addr = server.accept()              # 接电话（会卡住一直等）
print(f'接通了！对方地址: {addr}')

data = client.recv(1024)                    # 听对方说话（最多 1024 字节）
print(f'收到: {data}')

client.send(b'Roger that, over')            # 回复（b'' 表示字节）
client.close()
server.close()
```

**终端 2（客户端，后跑）：**
```python
import socket

client = socket.socket()                    # 买电话
client.connect(('127.0.0.1', 12345))       # 拨号：打给 127.0.0.1 分机 12345
client.send(b'Hello from drone')            # 说话
data = client.recv(1024)                    # 听回复
print(f'收到回复: {data}')
client.close()
```

**验证标准：** 终端 1 显示 `收到: b'Hello from drone'`，终端 2 显示 `收到回复: b'Roger that, over'`。

> `127.0.0.1` 的含义：永远指向你自己的电脑。同一台机器上的两个程序通信用这个地址。

### 1.5 语法详解

这一节按照**API 文档的标准格式**，逐个函数讲清楚：函数签名、参数类型、返回值类型、所有合法取值。

#### 1.5.1 `socket.socket()`

**函数签名：**
```python
socket.socket(family=AF_INET, type=SOCK_STREAM, proto=0, fileno=None)
```

**参数：**

| 参数 | 类型 | 是否必填 | 合法值 | 默认值 |
|------|------|---------|--------|--------|
| `family` | `int` | 否 | `socket.AF_INET`（IPv4）、`socket.AF_INET6`（IPv6） | `AF_INET` |
| `type` | `int` | 否 | `socket.SOCK_STREAM`（TCP）、`socket.SOCK_DGRAM`（UDP） | `SOCK_STREAM` |
| `proto` | `int` | 否 | `0`（自动选择协议），一般不填 | `0` |
| `fileno` | `int` 或 `None` | 否 | 不填 | `None` |

**返回值：** `socket` 对象（一个新创建的套接字）

**常用调用方式：**

```python
# TCP 套接字（最常用）
s = socket.socket()                                  # 全部默认
s = socket.socket(socket.AF_INET, socket.SOCK_STREAM) # 完整写法

# UDP 套接字（给 QGC 发数据时用）
s = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)

# IPv6
s = socket.socket(socket.AF_INET6, socket.SOCK_STREAM)
```

#### 1.5.2 `bind(address)`

**函数签名：**
```python
socket.bind(address)
```

**参数：**

| 参数 | 类型 | 格式 | 例子 |
|------|------|------|------|
| `address` | `tuple` | `(host: str, port: int)` | `('0.0.0.0', 5760)` |

- `host`：`str` 类型。你要监听的 IP 地址
- `port`：`int` 类型。你要监听的端口号，取值范围 **0 ~ 65535**

**`host` 的取值和含义：**

| 值 | 含义 |
|------|------|
| `'0.0.0.0'` | 监听本机**所有**网卡的所有 IP（公网、内网、本地回环都收） |
| `'127.0.0.1'` | 只监听本地回环，外网连不进来（适合本地测试） |
| `'8.152.207.82'` | 只监听这个特定 IP 的连接 |

**`port` 的取值范围：**

| 范围 | 用途 |
|------|------|
| 0 ~ 1023 | 系统保留（22=SSH, 80=HTTP, 443=HTTPS），需要 root 权限才能绑 |
| 1024 ~ 49151 | 用户可用，你的程序随便选 |
| 49152 ~ 65535 | 临时端口，操作系统自动分配给客户端 |

> 选端口时避开已占用的端口。Linux 上 `ss -tlnp` 可以看哪些端口已经被用了。

**返回值：** `None`（没有返回值，绑定失败会抛异常）

**常见异常：**

| 异常 | 原因 |
|------|------|
| `OSError: [Errno 98] Address already in use` | 端口被其他程序占用 |
| `OSError: [Errno 13] Permission denied` | 绑了 1024 以下的端口但没有 root 权限 |

#### 1.5.3 `listen(backlog)`

**函数签名：**
```python
socket.listen(backlog)
```

**参数：**

| 参数 | 类型 | 合法值 | 含义 |
|------|------|--------|------|
| `backlog` | `int` | `>= 0` 的整数 | 操作系统内部连接队列的最大长度 |

**`backlog` 可以取哪些值：**

```python
server.listen(0)    # 合法，队列长度由操作系统决定最小值（通常被自动设为某个默认值）
server.listen(1)    # 合法，队列最多排 1 个
server.listen(2)    # 合法，队列最多排 2 个
server.listen(5)    # 合法，常用值
server.listen(128)  # 合法，很多 Web 服务器的默认值
server.listen(-1)   # 报错！OSError: [Errno 22] Invalid argument
```

`listen(2)` 完全合法，意思是操作系统内部最多帮你排队 2 个人。

**到底应该设多少？**

| 场景 | 建议值 | 原因 |
|------|--------|------|
| 调试阶段 | 5 | 够用 |
| 生产环境（少量连接） | 5-10 | 撑死同时来 10 架飞机 |
| 生产环境（大量连接） | 128+ | Web 服务器标准 |

**返回值：** `None`

---

**`backlog` 到底控制什么——详细流程：**

TCP 连接建立是一个"三次握手"的过程（操作系统自动完成，你不用管）。握手完成后，连接进入一个队列，等你的程序调用 `accept()` 把它拿走。

```
飞机 connect() ──→ 三次握手（OS自动处理）──→ 进入 backlog 队列 ──→ 你的 accept() 拿走
                                                    │
                                          如果队列满了 → 新连接被拒绝（飞机端 connect 失败）
```

示意：

```
backlog = 2 的情况：

时刻 1: 飞机A 完成握手 → 进入队列  [A]          队列长度 1
时刻 2: 飞机B 完成握手 → 进入队列  [A, B]       队列长度 2（满了）
时刻 3: 飞机C 完成握手 → 队列满了 → C 被拒绝！
时刻 4: 你的程序 accept() → 拿走 A   [B]         队列长度 1（有空位了）
时刻 5: 飞机C 重试握手 → 进入队列    [B, C]      队列长度 2
```

**关键点：** `backlog` 没用完 ≠ 你没调用 `accept()`。只要你的 `accept()` 循环够快（拿到连接立刻开线程回去继续 accept），队列几乎永远是空的。所以 `listen(1)` 和 `listen(5)` 在实际使用中几乎没区别——队列永远是空的时候，设 1 还是设 5 都一样。

#### 1.5.4 `accept()`

**函数签名：**
```python
socket.accept()
```

**参数：** 无

**返回值：** `tuple[ socket, tuple[str, int] ]` — 一个包含两个元素的元组：

| 返回值元素 | 类型 | 含义 | 变量名习惯 |
|-----------|------|------|-----------|
| 第一个 | `socket` | 一个新的 socket 对象，专门用于和这个客户端收发数据 | `client` 或 `conn` |
| 第二个 | `tuple[str, int]` | 客户端的地址，`(IP地址, 端口号)` | `addr` 或 `address` |

**解包赋值：**
```python
client, addr = server.accept()
# client  →  socket 对象，用来 recv()/send()
# addr    →  ('127.0.0.1', 52431)，对方的 IP 和端口
```

也可以用下标：
```python
result = server.accept()
result[0]  #  client socket
result[1]  #  ('127.0.0.1', 52431)
```

**阻塞行为：** `accept()` 在没有连接到来时会**一直等待（阻塞）**，程序卡在这一行不动。直到有人 `connect()`，它才返回。

**程序验证——阻塞的效果：**
```python
import socket
import time

s = socket.socket()
s.bind(('0.0.0.0', 12345))
s.listen(5)

print('调用 accept()，程序将卡在这里...')
t0 = time.time()
client, addr = s.accept()         # ← 卡住，等别人 connect
t1 = time.time()

print(f'等了 {t1-t0:.1f} 秒后，有人连上来了')
print(f'对方地址: {addr}')
```

先在终端 1 跑上面这段——它会在 `accept()` 卡住。然后随便过几秒，在终端 2 跑 `nc 127.0.0.1 12345`，终端 1 立刻打印出等待时间。

#### 1.5.5 `recv(bufsize)`

**函数签名：**
```python
socket.recv(bufsize)
```

**参数：**

| 参数 | 类型 | 合法值 | 含义 |
|------|------|--------|------|
| `bufsize` | `int` | `> 0` 的整数（通常用 2 的幂：1024, 4096, 8192） | 单次最多接收的字节数 |

**返回值：** `bytes` — 接收到的数据

| 返回值 | 含义 |
|--------|------|
| `bytes` 对象，长度 > 0 | 收到了数据 |
| `b''`（长度为 0 的空字节串） | **对端已关闭连接**（对方调了 `close()` 或断网了） |

**`bufsize` 怎么选：**

| 值 | 适用场景 |
|------|---------|
| 1024 | 小数据量，每包几个字节到几百字节 |
| 4096 | MAVLink 场景（单帧最大 280 字节，4096 完全够） |
| 8192 | 一般网络编程的常用默认值 |
| 65536 | 大文件传输 |

MAVLink v2 单帧最大 267 字节，所以 `bufsize=4096` 绰绰有余。

**`recv()` 返回的数据不一定是你想要的长度：**

```python
# 对端: send(b'ABCDEFGHIJ')  发送 10 字节
# 你这端:
data = sock.recv(4)
print(data)   # b'ABCD'  ← 只拿到了 4 字节，剩下的还在操作系统缓冲区里！
data = sock.recv(4)
print(data)   # b'EFGH'  ← 又拿了 4 字节
data = sock.recv(4)
print(data)   # b'IJ'    ← 最后 2 字节
```

这就是**流式传输**的特点——TCP 不保留消息边界。你不能假设一次 `recv()` 刚好拿到一个完整消息。这就是第四节要讲的「粘包处理」。

**阻塞行为：** 如果缓冲区为空且对面还没关连接，`recv()` 会阻塞等待。如果想不阻塞，可以用 `sock.setblocking(False)` 或 `sock.settimeout(秒数)`。

#### 1.5.6 `send(data)`

**函数签名：**
```python
socket.send(data)
```

**参数：**

| 参数 | 类型 | 合法值 | 含义 |
|------|------|--------|------|
| `data` | `bytes` | 任意 bytes 对象 | 要发送的数据 |

> **不能传 `str`（字符串）！** 必须是 `bytes`。如果要发送字符串，先 `.encode()`：
> ```python
> sock.send('你好'.encode('utf-8'))
> ```

**返回值：** `int` — 实际发送的字节数

```python
sent = sock.send(b'Hello')
print(sent)   # 5  （成功发送了 5 字节）

sent = sock.send(b'Hello')
print(sent)   # 可能是 3  （只发出了 3 字节，网络忙）
```

> 大多数情况下返回值等于你要发的字节数。但在网络很忙时，可能只发出去一部分。**生产环境下应该用 `sendall()` 代替 `send()`**，它会自动循环直到全部发完：
> ```python
> sock.sendall(b'Hello')   # 保证全部发完，无返回值
> ```

#### 1.5.7 `connect(address)`

**函数签名：**
```python
socket.connect(address)
```

**参数：**

| 参数 | 类型 | 格式 | 例子 |
|------|------|------|------|
| `address` | `tuple` | `(host: str, port: int)` | `('8.152.207.82', 5760)` |

**返回值：** `None`

**阻塞行为：** 连接建立成功后才返回。如果对方没在监听，抛异常：

| 异常 | 原因 |
|------|------|
| `ConnectionRefusedError: [Errno 111] Connection refused` | 目标端口没有人在监听 |
| `socket.timeout: timed out` | 连接超时（网络不通、防火墙拦截） |

**客户端不需要 `bind()`：** 操作系统会自动给客户端分配一个临时端口（比如 52341），你不需要手动指定。

**使用示例：**
```python
sock = socket.socket()
sock.settimeout(5)  # 设置 5 秒超时

try:
    sock.connect(('8.152.207.82', 5760))
    print('连接成功')
except ConnectionRefusedError:
    print('服务器没开')
except socket.timeout:
    print('连接超时')
```

#### 1.5.9 `setsockopt(level, optname, value)`

**函数签名：**
```python
socket.setsockopt(level, optname, value)
```

**参数：**

| 参数 | 类型 | 含义 | 常用值 |
|------|------|------|--------|
| `level` | `int` | 选项的协议层级 | `socket.SOL_SOCKET`（socket 层选项） |
| `optname` | `int` | 具体要设置的选项 | `socket.SO_REUSEADDR`（允许端口重用） |
| `value` | `int` | 选项值 | `1`（启用）、`0`（禁用） |

**最常用的调用（调试必备）：**
```python
server.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
```

**作用：** 允许在程序关闭后**立刻**重新绑定同一个端口，不等操作系统的 TIME_WAIT 超时（默认要等 1-2 分钟）。调试时反复重启程序没这一行必报错 `Address already in use`。

**返回值：** `None`

#### 1.5.10 常用 socket 选项速查

这些都是在 `setsockopt` 里用的。按你需要什么功能来查。

**调试必备（每个 socket 都加）：**

| 选项 | level | optname | value | 作用 |
|------|-------|---------|-------|------|
| 端口重用 | `SOL_SOCKET` | `SO_REUSEADDR` | `1` | 关了立刻能重新 bind，不等 TIME_WAIT |
| 地址重用 | `SOL_SOCKET` | `SO_REUSEPORT` | `1` | Linux 3.9+，允许多个进程绑同一个端口做负载均衡 |

```python
sock.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
```

**超时控制：**

| 你要干嘛 | 方法 | 参数 |
|---------|------|------|
| 收发数据超时 | `sock.settimeout(秒)` | float，秒 |
| 获取当前超时 | `sock.gettimeout()` | 返回 float 或 `None` |

```python
sock.settimeout(5.0)      # 5 秒后没收到数据就抛 socket.timeout

```

---

**TCP 行为控制：**

| 选项 | level | optname | value | 作用 |
|------|-------|---------|-------|------|
| 禁用 Nagle 算法 | `IPPROTO_TCP` | `TCP_NODELAY` | `1` | 数据立刻发，不等凑满一块。实时性要求高的场景（比如你的飞机数据）强烈建议开 |
| 心跳检测 | `SOL_SOCKET` | `SO_KEEPALIVE` | `1` | TCP 定期发心跳包，检测对端是否还活着。配合下面三个用 |
| 心跳空闲时间 | `IPPROTO_TCP` | `TCP_KEEPIDLE` | `60` | 60 秒没数据就开始发心跳探测 |
| 心跳探测间隔 | `IPPROTO_TCP` | `TCP_KEEPINTVL` | `10` | 每次探测间隔 10 秒 |
| 心跳探测次数 | `IPPROTO_TCP` | `TCP_KEEPCNT` | `3` | 连续 3 次探测没回应 → 判定断开 |

```python
# 实时传输：数据来了立刻发，不等
sock.setsockopt(socket.IPPROTO_TCP, socket.TCP_NODELAY, 1)

# 4G 弱网场景：60 秒无数据就开始探测，避免僵尸连接
sock.setsockopt(socket.SOL_SOCKET, socket.SO_KEEPALIVE, 1)
sock.setsockopt(socket.IPPROTO_TCP, socket.TCP_KEEPIDLE, 60)
sock.setsockopt(socket.IPPROTO_TCP, socket.TCP_KEEPINTVL, 10)
sock.setsockopt(socket.IPPROTO_TCP, socket.TCP_KEEPCNT, 3)
```

> **Nagle 算法是什么？** TCP 默认会等数据凑够一定大小再发出去（为了节省网络带宽）。`TCP_NODELAY` 禁掉这个行为，数据立刻发。MAVLink 包很小（几十到几百字节），等不得，必须开 `TCP_NODELAY`。

---

**缓冲区大小：**

| 选项 | level | optname | value | 作用 |
|------|-------|---------|-------|------|
| 接收缓冲区 | `SOL_SOCKET` | `SO_RCVBUF` | 字节数 | 操作系统收数据的缓冲区大小 |
| 发送缓冲区 | `SOL_SOCKET` | `SO_SNDBUF` | 字节数 | 操作系统发数据的缓冲区大小 |

```python
sock.setsockopt(socket.SOL_SOCKET, socket.SO_RCVBUF, 65536)  # 64KB 接收缓冲
sock.setsockopt(socket.SOL_SOCKET, socket.SO_SNDBUF, 65536)  # 64KB 发送缓冲
```

---

**阻塞/非阻塞：**

| 你要干嘛 | 方法 | 参数 |
|---------|------|------|
| 设为非阻塞 | `sock.setblocking(False)` | bool |
| 查询当前 | `sock.getblocking()` | 返回 bool |

```python
sock.setblocking(False)    # recv() 没数据立刻抛 BlockingIOError，不等
sock.setblocking(True)     # 默认行为，recv() 没数据就死等

# 非阻塞模式下的 recv:
try:
    data = sock.recv(4096)
except BlockingIOError:
    # 当前没数据，先去干别的事再来
    pass
```

**选项层级对照表：**

| level 常量 | 范围 | 用在哪 |
|-----------|------|-------|
| `socket.SOL_SOCKET` | 所有 socket 通用 | `SO_REUSEADDR`、`SO_KEEPALIVE`、`SO_RCVBUF` |
| `socket.IPPROTO_TCP` | 仅 TCP socket | `TCP_NODELAY`、`TCP_KEEPIDLE` |
| `socket.IPPROTO_IP` | 仅 IPv4 socket | `IP_TTL` |

## 2. 第一步：TCP Server — 接收数据

## 3. 第二步：理解 MAVLink 协议

### 3.1 MAVLink 是什么

MAVLink 是**飞控对外通信的语言**。ArduPilot 和 PX4 都用它。你的 DM-FC01 飞控通过串口（TELEM1）不停往外吐 MAVLink 数据包。

### 3.2 MAVLink 数据包长什么样

MAVLink v2 一个包的结构：

```
字节位置    内容                    举例(心跳包)
─────────────────────────────────────────────
[0]       帧头 STX                0xFD  (固定值，v2 标识)
[1]       负载长度 LEN            0x09  (9 字节)
[2]       不兼容标志              0x00
[3]       兼容标志                0x00
[4]       包序号 SEQ               0x00  (0-255，循环递增)
[5]       系统 ID SYSID           0x01  (飞机编号 1-255)
[6]       组件 ID COMPID          0x01  (1=飞控)
[7]       消息 ID (低字节)         0x00  │
[8]       消息 ID (中字节)         0x00  ├─ MSGID = 0 (HEARTBEAT)
[9]       消息 ID (高字节)         0x00  │
[10-18]   负载 (9 字节)            ...   (消息内容)
[19]      校验 (低字节)            XX    │
[20]      校验 (高字节)            XX    ├─ CRC16 校验
─────────────────────────────────────────────
总长度 = 10 + LEN + 2 = 12 + LEN 字节
最小包 = 12 字节 (LEN=0)，最大包 = 267 字节 (LEN=255)
```

### 3.3 常见的消息类型

| 消息 ID | 名称 | 内容 | 频率 |
|---------|------|------|------|
| 0 | HEARTBEAT | 飞机类型、飞控类型、状态 | 1 Hz |
| 1 | SYS_STATUS | 电池电压、电流、剩余电量 | 1 Hz |
| 30 | ATTITUDE | 横滚/俯仰/偏航角 | 5-10 Hz |
| 33 | GLOBAL_POSITION_INT | GPS 经纬度、高度 | 3-5 Hz |

### 3.4 动手：手动构造一个心跳包

心跳包（HEARTBEAT）的负载是 9 个字节：

| 字段 | 类型 | 值 | 含义 |
|------|------|-----|------|
| type | uint8 | 2 | 四旋翼 |
| autopilot | uint8 | 3 | ArduPilot |
| base_mode | uint8 | 129 | 已解锁+自定义模式 |
| custom_mode | uint32 | 0 | 模式编号 |
| system_status | uint8 | 4 | 活跃 |
| mavlink_version | uint8 | 3 | MAVLink v2 |

用 Python 构造这 9 个字节：

## 4. 第三步：解析 MAVLink 帧

### 4.1 核心难题：粘包

TCP 是**流式传输**。你发三个包，TCP 可能把它们混在一起发过来：

```
你发送:   [包A] [包B] [包C]
TCP 可能: [包A前半] ... [包A后半+包B] ... [包C]
```

所以接收端需要自己**从字节流中切出完整的包**。

### 4.2 解决方案：缓冲区 + 帧查找

```
新数据来了 → 追加到缓冲区 → 找帧头 0xFD → 
读 LEN 字段 → 算出完整帧长度 → 
等缓冲区够长 → 切出完整帧 → 
验证 CRC → 通过！处理这一帧
```

### 4.3 实现骨架

```python
def find_frame(data):
    """
    从字节流中找到第一个完整 MAVLink 帧。
    返回 (frame_bytes, remaining_bytes)
    如果没找到完整帧，返回 (None, data)
    """
    # 1. 找帧头 0xFD
    idx = data.find(b'\xfd')
    if idx == -1:
        return (None, data)     # 没找到帧头
    if idx > 0:
        data = data[idx:]       # 丢弃帧头前的垃圾
    
    # 2. 至少要有 10 字节帧头
    if len(data) < 10:
        return (None, data)
    
    # 3. 读 payload 长度（在 byte[1]）
    payload_len = data[1]
    frame_len = 10 + payload_len + 2   # 帧头(10) + 负载 + CRC(2)
    
    # 4. 缓冲区够不够一个完整帧？
    if len(data) < frame_len:
        return (None, data)     # 半包，继续等
    
    frame = data[:frame_len]
    remaining = data[frame_len:]
    
    # 5. TODO: 验证 CRC
    # 验证通过返回 frame，失败则跳过一字节重试
    return (frame, remaining)
```

### 4.4 关键点

- **`None` 返回** = 数据不够，需要等更多数据（`recv()` 再收一些追加到缓冲区）
- **有返回值** = 切出一个完整帧，处理完继续循环切下一个
- **校验失败** = 跳过 1 字节，从下一个可能位置继续找

## 5. 第四步：转发给 QGC

### 5.1 QGC 怎么接收数据

QGroundControl (QGC) 默认监听 **UDP 14550 端口**。你把 MAVLink 包通过 UDP 发到这个端口，QGC 就能看到飞机。

### 5.2 UDP vs TCP

| | TCP | UDP |
|------|-----|-----|
| 类比 | 打电话（先接通再说话） | 发快递（直接扔出去，不管对方收没收到） |
| 可靠性 | 保证送达 | 可能丢 |
| 速度 | 较慢（有握手、确认） | 更快 |
| 用在哪 | 飞机 → 服务器（4G 链路） | 服务器 → QGC（局域网/本机） |
| Python | `socket.SOCK_STREAM` | `socket.SOCK_DGRAM` |

### 5.3 UDP 发送示例

```python
import socket

qgc_sock = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)

# 发送数据给 QGC
qgc_sock.sendto(mavlink_frame_bytes, ('127.0.0.1', 14550))
```

就一行。不需要 `connect()`，不需要 `listen()`。

### 5.4 路由器逻辑

```
TCP 收字节 → 切出 MAVLink 帧 → 原封不动 → UDP 发给 QGC
```

**关键：不需要解析 MAVLink 内容。** 路由器只管"原封不动转发"，不解包、不修改。

### A. TCP Server 框架

```python
import socket
import threading

class TCPServer:
    """TCP 服务器 — 接收飞机 4G 连接"""
    
    def __init__(self, host='0.0.0.0', port=5760):
        self.host = host
        self.port = port
        self.handlers = []          # 存储每个连接的处理函数
    
    def on_connect(self, handler):
        """注册回调：有人连上来时调用 handler(client, addr)"""
        self.handlers.append(handler)
    
    def start(self):
        sock = socket.socket()
        sock.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
        sock.bind((self.host, self.port))
        sock.listen(10)
        print(f"[TCP] 监听 {self.host}:{self.port}")
        
        while True:
            client, addr = sock.accept()
            # 每个连接一个线程
            t = threading.Thread(
                target=self._handle_client,
                args=(client, addr),
                daemon=True
            )
            t.start()
    
    def _handle_client(self, client, addr):
        print(f"[+] 新连接: {addr}")
        try:
            for handler in self.handlers:
                handler(client, addr)
        except Exception as e:
            print(f"[-] 连接异常: {e}")
        finally:
            client.close()

# 使用示例
server = TCPServer(port=5760)
server.on_connect(lambda client, addr: print(f"飞机 {addr} 连上了!"))
server.start()
```

### C. CRC 校验

```python
def mavlink_crc16(data: bytes, crc_extra: int) -> int:
    """MAVLink CRC-16/MCRF4XX"""
    crc = 0xFFFF
    for b in bytes([crc_extra & 0xFF]) + data:
        crc ^= b
        for _ in range(8):
            if crc & 1:
                crc = (crc >> 1) ^ 0x8408
            else:
                crc >>= 1
    return crc & 0xFFFF

def verify_crc(frame: bytes) -> bool:
    """验证 MAVLink v2 帧的 CRC"""
    msgid = frame[7] | (frame[8] << 8) | (frame[9] << 16)
    crc_extra = CRC_EXTRA.get(msgid, 0)
    
    payload_len = frame[1]
    data_end = 10 + payload_len    # 从 byte[1] 到 payload 末尾
    data = frame[1:data_end]
    
    expected = mavlink_crc16(data, crc_extra)
    actual = frame[data_end] | (frame[data_end + 1] << 8)
    return expected == actual
```

```python
import sqlite3, json, time

class DroneDB:
    def __init__(self, path='drone.db'):
        self.conn = sqlite3.connect(path, check_same_thread=False)
        self._init_tables()
    
    def _init_tables(self):
        self.conn.execute('''CREATE TABLE IF NOT EXISTS drones (
            sysid INTEGER PRIMARY KEY, name TEXT,
            first_seen TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
            last_seen TIMESTAMP DEFAULT CURRENT_TIMESTAMP
        )''')
        self.conn.execute('''CREATE TABLE IF NOT EXISTS telemetry (
            id INTEGER PRIMARY KEY AUTOINCREMENT,
            drone_id INTEGER, msg_type TEXT,
            lat REAL, lon REAL, alt REAL, battery REAL,
            raw TEXT, time TIMESTAMP DEFAULT CURRENT_TIMESTAMP
        )''')
        self.conn.commit()
    
    def update_drone_seen(self, sysid: int):
        self.conn.execute(
            'INSERT OR REPLACE INTO drones (sysid, name, first_seen, last_seen) '
            'VALUES (?, ?, COALESCE((SELECT first_seen FROM drones WHERE sysid=?), CURRENT_TIMESTAMP), CURRENT_TIMESTAMP)',
            (sysid, f'drone-{sysid}', sysid)
        )
        self.conn.commit()
    
    def save_telemetry(self, drone_id: int, msg_type: str, **kwargs):
        self.conn.execute(
            'INSERT INTO telemetry (drone_id, msg_type, lat, lon, alt, battery, raw) '
            'VALUES (?,?,?,?,?,?,?)',
            (drone_id, msg_type,
             kwargs.get('lat'), kwargs.get('lon'), kwargs.get('alt'),
             kwargs.get('battery'), json.dumps(kwargs))
        )
        self.conn.commit()
    
    def get_drones(self) -> list:
        rows = self.conn.execute(
            'SELECT * FROM drones ORDER BY last_seen DESC'
        ).fetchall()
        return [{'sysid': r[0], 'name': r[1], 'last_seen': r[3]} for r in rows]
```
