# 无人机仿真开发环境 — 从零搭建指南

> 给团队成员的快速上手指南 | 预计耗时：首次安装 40 分钟，之后每次启动 30 秒

---

## 这是什么？

在电脑上模拟一架**完整的无人机飞控**，配合地面站软件，可以在不炸机、不花钱、不出门的情况下：

- ✈️ 测试所有飞行指令（起飞、降落、返航、航点任务）
- 🎮 切换所有飞行模式（增稳/悬停/特技/自动）
- 📡 实时查看飞机遥测数据（姿态、GPS、电池、传感器）
- 🔧 调试舵机、云台、投放舱等外设指令
- 🎯 规划航点任务，模拟完整飞行流程
- 🖥️ 3D 场景中观察飞机实时姿态和舵面动作

**和真飞控的关系**：SITL 运行的固件代码 = 真飞控固件代码，参数表、指令响应、模式切换完全一致。在这里测通 → 上真飞机直接飞。

## 第一步：安装 WSL + Ubuntu 22.04

**WSL** = Windows Subsystem for Linux，让你在 Windows 里跑 Linux。

### 1.1 启用 WSL（只需一次，管理员 PowerShell）

右键开始菜单 → **终端(管理员)**，粘贴：

```powershell
wsl --install -d Ubuntu-22.04
```

等它下载安装完，会自动弹出 Ubuntu 窗口，创建你的账号：
- 用户名：`你的名字拼音`
- 密码：`设一个好记的`（输入时不显示，正常）

### 1.2 验证安装

开始菜单搜索 `Ubuntu-22.04`，打开应该看到绿色提示符 `用户名@机器名:~$`，这就成了。

---

## 第二步：安装 ArduPilot SITL

**SITL** = Software In The Loop，在电脑上运行的飞控固件，行为和真飞控一模一样。

在 Ubuntu-22.04 终端里**逐条**粘贴运行（每条跑完再跑下一条）：

```bash
# 1. 安装系统依赖
sudo apt update
sudo apt install -y python3-pip git

# 2. 安装 Python 依赖
pip3 install --user empy==3.3.4 pexpect pymavlink mavproxy future

# 3. 确保 pip 安装的命令可用
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc

# 4. 克隆 ArduPilot 源码（约 300MB，等几分钟）
cd ~
git clone --depth 1 https://github.com/ArduPilot/ardupilot.git
cd ardupilot
git submodule update --init --recursive
```

### 验证安装

```bash
cd ~/ardupilot/ArduCopter
../Tools/autotest/sim_vehicle.py -v ArduCopter --console --map
```

第一次会编译固件（约 2 分钟），看到 `ArduPilot Ready` 就成功了。

---

## 第三步：安装地面站

### 3.1 QGroundControl（主要使用，推荐）

1. 下载：https://github.com/mavlink/qgroundcontrol/releases/latest
2. 选 `QGroundControl-installer.exe`
3. 双击安装，一路点下一步
4. 桌面出现 QGC 图标

**为什么用 QGC**：UI 现代、中文好、自动发现飞控/SITL、无需手动配端口。

### 3.2 Mission Planner（备用，高级调试）

1. 下载：https://firmware.ardupilot.org/Tools/MissionPlanner/MissionPlanner-latest.zip
2. 解压到任意位置
3. 双击 `MissionPlanner.exe`

**为什么保留 MP**：内置 SITL（不需要 WSL 就能仿真）、日志分析最强、参数配置最全。

---

## 第四步：可选——3D 仿真

### FlightGear（推荐，轻量）

在 Ubuntu-22.04 终端里：

```bash
sudo apt install -y flightgear
```

然后启动 3D 模式：

```bash
cd ~/ardupilot/ArduCopter
export PATH="$HOME/.local/bin:$PATH"
../Tools/autotest/sim_vehicle.py -v ArduCopter -f flightgear --console --map
```

FlightGear 弹出 3D 窗口后能看到飞机实时姿态和舵面动作。

---

## 第五步：跑通全流程

### 日常启动（三种方式任选）

**方式A — WSL 终端直接启动**

```bash
cd ~/ardupilot/ArduCopter
export PATH="$HOME/.local/bin:$PATH"
../Tools/autotest/sim_vehicle.py -v ArduCopter --console --map
```

**方式B — 双击脚本启动**

双击项目里的 `启动SITL.bat`，输密码，选模式。

**方式C — Mission Planner 自带 SITL**（最快，无需 WSL）

打开 MP → 菜单 Simulation → 选 Quadcopter → Start

### 连接地面站

SITL 启动后：
- **QGC** — 自动发现，左上角弹出飞机图标，点一下就连上
- **MP** — 右上角选 UDP，端口 14550，点 Connect

## 常见问题

| 问题 | 解决 |
|------|------|
| `wsl --install` 报错 | 确保 BIOS 里开了虚拟化（Intel VT-x / AMD SVM），Windows 功能里开了"适用于 Linux 的 Windows 子系统"和"虚拟机平台" |
| `ModuleNotFoundError: No module named 'xxx'` | `pip3 install --user xxx` |
| QGC 不自动连接 SITL | 手动添加：Settings → Comm Links → Add → UDP，端口 14550 |
| SITL 启动后卡住 | 等 `ArduPilot Ready` 出现，第一次编译慢 |
| Ubuntu 终端乱码 | 不影响使用，忽略 |
