## 1. 电路设计与电控（秋森 & 一凡）

### 整体路径

```
电池组装 → 研究成品 BMS → 换电站整体架构 → 与飞机组对接需求 → 设计投放舱与降落伞电控
```

### 阶段一：电池组装（2-3 周）

**目标**：亲手组装一包能用的 4S 无人机电池。

**学习内容**：
- 锂离子电池基础知识（18650 vs 21700，串并联特性，内阻概念）
- 点焊技术（实操）—— 镍片厚度、点焊电流、焊点间距
- 平衡充电原理 —— 4S 充电器使用，每片电芯的电压范围（3.0V-4.2V）
- 电池保护板（BMS）接线 —— 这是理解 BMS 的第一步

**实操任务**：
1. 用万用表测量每节 21700 电压，记录并配对（电压相近的放一组）
2. 点焊镍片连接 4S 电池组
3. 焊接 BMS 保护板
4. 用 B6 充电器做一次完整充放电循环，记录容量

**经验预警**：
- ⚡ 点焊时**务必佩戴护目镜**，镍片短路瞬间发红太常见了
- ⚡ 焊接 BMS 时**先焊 B-（电池负极），再焊各节正极**，不然容易烧 BMS
- ⚡ 第一次充电在室外/阳台做，电池冒烟了你至少有时间跑
- ⚡ 不要用烙铁直接焊电池极片！必须用点焊机。烙铁高温会损伤电池内部结构

**推荐资源**：
- YouTube 搜索 "DIY 4S Li-ion battery pack"
- B站 "锂电老板" 系列视频
- 《锂电池基础知识》— 任何一本入门级电子书都行

---

### 阶段二：研究成品 BMS（2-3 周）

**目标**：理解 BMS 的工作原理，能判断现在的 BMS 是否够用。

**学习内容**：
- BMS 核心功能：过充保护、过放保护、过流保护、短路保护、均衡充
- 被动均衡 vs 主动均衡（被动靠电阻放电耗掉多余电量，主动用电感/电容转移电量）
- 关键参数：持续放电电流、瞬间峰值电流、均衡电流
- 为什么用户手册标注的 BMS "建议后续更换"？

**实操任务**：
1. 把你手上这块 ¥21.50 的 BMS 拿出来，查芯片型号，找到 datasheet
2. 对比无人机飞行的典型电流需求（你的电机还没定，先按单电机 15-20A 估算，4 个就是 60-80A）
3. 如果 BMS 持续放电只有 60A，飞行时可能触发保护 → 炸机
4. 调研替代方案：可以不经过 BMS 放电（飞控有电压监测），BMS 只负责充电保护

**经验预警**：
- 🔥 无人机行业的常见做法是：**电池直连飞控/电调 → 飞控监测电压 → 低电压返航**。BMS 的过放保护反而可能在飞行中误切断电源
- 🔥 保护板标称 100A 不代表持续 100A，看 datasheet 里的 continuous discharge current

---

### 阶段三：换电站整体架构（4-6 周）

**目标**：设计换电站的电气架构方案。

**学习内容**：
- 换电站功能分解：电池充电槽、电池定位检测、电池锁止/解锁机构、充电管理
- 机械和电气的接口：无人机降落 → 电池定位 → 解锁旧电池 → 取出 → 放入新电池 → 锁止 → 检测到位
- 换电站控制核心：一块 STM32/ESP32 做主控，管理充电、检测、通信
- 充电策略：多路独立充电（每个槽位一个充电模块）还是集中充电 + 切换

**实操任务**：
1. 画换电站的电气框图（纸笔即可，再进 CAD）
2. 列出需要的传感器：限位开关（检测电池到位）、电流传感器（充电状态）、温度传感器（安全）
3. 调研充电模块：用成品 B6 充电器改装还是 TP4056 模块自己搭
4. 思考：换电站怎么和无人机通信？（CAN？串口？WiFi？4G 中台？）

**经验预警**：
- 🔧 换电站最难的永远是**机械对准**而非电气。弹簧触点的接触电阻、寿命、对准容差，这些比电路设计头疼十倍
- 🔧 第一阶段图纸上先画好，不要急着买件。和建模组秋森（你自己）紧密沟通机械结构
- 🔧 建议先做一个单槽位的充电站验证，再扩展到多槽位

---

### 阶段四：投放舱与降落伞电控（3-4 周）

**目标**：设计和实现投放舱和降落伞的电子控制系统。

**学习内容**：
- 舵机控制基础（PWM 信号，50Hz，0.5ms-2.5ms 对应 0°-180°）
- 投放舱：一个舵机控制舱门开合，需要限位检测（防止堵转烧舵机）
- 降落伞：需要一个可靠触发机制 —— 常见方案是用电热丝熔断尼龙绳，或电磁锁扣
- 控制信号来源：飞控的 AUX PWM 通道（飞控检测到失控/低电压 → 自动输出信号）或独立单片机判断

**实操任务**：
1. 买一个 SG90 舵机（¥5），用 Arduino/STM32 输出 PWM，控制它转
2. 设计投放舱的开关逻辑：收到指令 → 舵机转 90° → 舱门打开 → 延时 → 舵机归位
3. 降落伞触发测试：用电热丝 + 尼龙绳做一次地面触发实验（**绝对不要在飞着的飞机上第一次测试！**）
4. 与飞机组对接：飞控能提供几个空闲 PWM 通道？信号协议是什么？

**经验预警**：
- 🪂 降落伞触发**不能依赖单一信号**。至少要有飞控失联 + 手动遥控两个触发源
- 🪂 舵机在堵转时电流可达 1-2A！要独立供电，不要从飞控 5V BEC 取电
- 🪂 投放舱的机械设计（舱门铰链位置、舵机臂长度、重力方向）比电控复杂，先和建模组沟通

## 2. 飞控与飞机平台（馨淼 & 哲宇）

> 📡 **配套学习笔记**：`飞机层-学习笔记.md` — 飞机层完整学习笔记（飞控四层子结构、四种控制方式、MAVLink 指令到 PWM 全路径、Failsafe 安全底线、ArduPilot 源码导航、自检清单）。建议学完阶段二（刷写固件）后阅读。

### 整体路径

```
研究现有飞机 → 尝试刷写固件 → 了解固件各方面功能与底层代码 → 加装代码和模块实现新功能
```

### 阶段一：研究现有飞机（2 周）

**目标**：理解多旋翼无人机的组成部分和每部分的作用。

**学习内容**：
- 四旋翼基本组成：机架、飞控、电调、电机、桨、电池、接收机
- 飞行原理：四个电机差速控制 → 俯仰/横滚/偏航/油门
- PID 控制直觉理解（不要先啃公式）：
  - P（比例）：偏了多少，就修正多少
  - I（积分）：一直没修正到位，加大力度
  - D（微分）：变化太快了，刹车
- ArduPilot 和 PX4 的区别和选型

**实操任务**：
1. 看 B站 "ArduPilot 入门"系列视频（搜 UAV 飞行控制）
2. 用手拿着 DM-FC01，连上 QGroundControl，动一动它，看姿态显示跟着变
3. 用 USB 给飞控供电，打开 Mission Planner / QGC 的 MAVLink Inspector，看原始数据流

**经验预警**：
- 📡 飞控不装桨时接电池上电是安全的（没负载），但装了桨**必须在空旷地方、距离人 5 米以上**
- 📡 第一次连 QGC 大概率 USB 驱动没装好，Windows 下 Zadig 工具是救星
- 📡 DM-FC01 的 USB 不对外供电！接 GPS/接收机等外设时必须接电池

---

### 阶段二：刷写固件（1-2 周）

**目标**：学会给飞控刷固件，能切换 PX4 和 ArduPilot。

**学习内容**：
- 飞控固件的结构：Bootloader → 固件本体 → 参数存储区
- PX4 刷写：QGC → 固件页面 → USB 直连 → 选择 PX4 Flight Stack → 在线下载并烧录
- ArduPilot 刷写：Mission Planner → Install Firmware → 选择对应版本
- Bootloader 原理：按住 Boot 键 → 进入 DFU 模式 → STM32CubeProgrammer 烧录

**实操任务**：
1. 把飞控刷成 PX4 v1.16，连 QGC 确认成功
2. 再把飞控刷成 ArduPilot，连 Mission Planner 确认成功
3. 刷回 PX4，再来一遍。**刷 5 遍以上，你就不会再怕"变砖"了**
4. 故意刷一个错的固件（比如刷个 Plane 固件到 Copter 上），看会怎样

**经验预警**：
- 🔄 DM-FC01 支持双固件，但**刷固件前先把现有参数备份出来**！参数是无数人调出来的经验值
- 🔄 固件刷失败不要慌，飞控板子上有 Boot 按钮，按住再插 USB 就能进 DFU 模式强制刷
- 🔄 PX4 和 ArduPilot 的参数体系完全不同，从 PX4 切 ArduPilot 后需要从零校准加速度计、罗盘、电调

---

### 阶段三：了解固件功能与底层代码（持续进行）

**目标**：能看懂固件的模块结构，知道在哪加新功能。

**学习内容**：
- PX4 的 uORB 消息机制：每个传感器是一个 topic，每个控制模块订阅和发布 topic
- ArduPilot 的 HAL 层：将硬件操作抽象，上层代码不依赖具体 MCU 型号
- 关键模块位置（以 ArduPilot 为例）：
  - `libraries/AP_Mount/` — 云台控制
  - `libraries/AP_Camera/` — 相机触发
  - `ArduCopter/` — 多旋翼主逻辑
  - `libraries/RC_Channel/` — 遥控器通道处理

**实操任务**：
1. 克隆 ArduPilot 源码，不修改，先编译一遍（用用户手册里的 `./waf configure --board=DM-FC01 && ./waf copter`）
2. 找到飞控上 LED 灯的控制代码，改一下 LED 闪烁频率，重新编译烧录，确认生效
3. 在 `RC_Channel` 里加一个 AUX 通道的自定义功能（比如 AUX1 高电平 → 输出一个调试日志）
4. 阅读 ArduPilot 的 `mode.cpp`，理解飞行模式切换逻辑

**经验预警**：
- 💻 ArduPilot 代码库有 100 万行以上，不要试图全部读懂。**先找到你需要的模块，只读那个模块**
- 💻 PX4 更适合做研究性质的功能开发（模块化好），ArduPilot 更适合快速出产品（社区大，资料多）
- 💻 改代码前先确认两件事：① 硬件引脚没被占用 ② 不影响飞控核心实时性
- 💻 **ArduPilot 推荐为该项目默认固件** — 社区大、参数多、资料全、中文教程丰富

---

### 阶段四：加装代码和模块（后期）

**目标**：为飞机增加新功能——投放舱控制、降落伞触发、ArUco 码起降。

**学习内容**：
- 如何在 ArduPilot 中创建一个新 library（类比 AP_Camera 或 AP_Mount）
- 如何通过 MAVLink 自定义消息与树莓派/地面站通信
- 如何在飞控上添加一个新的传感器驱动

**实操任务**：
1. 在飞控上添加一个舵机输出通道（用于投放舱），通过遥控器 AUX 通道控制
2. 写一段代码：飞控检测到 `FS_THR_ENABLE` failsafe → 输出 PWM 信号触发降落伞
3. 接上外部 LED 或蜂鸣器，让它在特定条件下（低电压/失联）发出警示

**经验预警**：
- 🔌 飞控的 GPIO 是 3.3V 电平，不要直接驱动需要 5V 的外设
- 🔌 所有飞控上改装必须先在地面验证 30 分钟以上，确保不影响核心飞控功能
- 🔌 多余的 PWM 输出走 AUX 通道是最安全的做法——ArduPilot 已经帮你做好了硬件抽象

## 3. 通信与中台系统（哲宇 & 用户）

> 📡 **配套学习笔记**：`通讯层-学习笔记.md` — 通讯层完整学习笔记（链路拓扑、五种协议对比、分段回环测试、提问路线回顾、学习方法论）。建议读完本章后对照阅读。`通信链路与MAVLink协议学习笔记.md` 可作为快速参考。

### 整体路径

```
学习单片机 → 学习单片机与服务器通信 → 研究 4G 通信模块 → 为中台/换电站/飞机加装模块
```

### 阶段一：单片机入门（3-4 周）

**目标**：能用单片机做简单输入输出，理解串口通信。

**推荐平台**：ESP32（WiFi/蓝牙内置，¥30）或 STM32F103（经典入门，¥15）

**学习内容**：
- GPIO 输入输出（按键 → 点亮 LED）
- 串口通信 UART（单片机 ←→ 电脑通信）
- 定时器与 PWM（控制舵机/呼吸灯）
- ADC 采样（读取电池电压、温度传感器）
- ESP32 特有：WiFi 连接、HTTP 请求、MQTT 发布订阅

**实操任务（由浅入深）**：
1. 点亮一个 LED（点灯是嵌入式领域的"Hello World"）
2. 按下一个按键，LED 亮；松开，LED 灭
3. 通过串口向电脑每隔 1 秒发送一条 "Hello Drone"
4. 电脑通过串口发送 "1" → 点亮 LED；发送 "0" → 熄灭 LED
5. ESP32：连接 WiFi，访问 `http://httpbin.org/get` 并打印返回内容
6. ESP32：连接到 MQTT broker，发布一条消息到 topic `drone/test`

**经验预警**：
- 🔌 **第一坑永远是驱动**：Windows 上 ESP32 的 CP210x 驱动、STM32 的 ST-Link 驱动，提前装好
- 🔌 **第二坑是波特率**：串口通信两端波特率不一致 → 乱码，浪费 2 小时算少的
- 🔌 **第三坑是供电**：从单片机 USB 口取电驱动舵机 → 电压不够 → 舵机不动 → 以为是代码问题
- 🔌 ESP32 的 GPIO 是 3.3V！接 5V 传感器会烧

**推荐教程**：
- B站 "ESP32 入门教程"（太极创客）—— 中文，零基础友好
- Arduino IDE 开发 ESP32 —— 环境搭建最简单，先跑起来再说

---

### 阶段二：单片机与服务器通信（3-4 周）

**目标**：理解网络通信协议，单片机能跟服务器双向交互。

**学习内容**：
- TCP vs UDP：TCP 可靠但慢，UDP 快但会丢包。实时控制 → UDP；状态上报 → 可以 TCP
- MQTT 协议：物联网最主流的发布/订阅协议，特别适合多设备通信
- HTTP/HTTPS：用于非实时请求（比如上传飞行日志）
- 串口透传：单片机收到服务器消息 → 通过串口转发给飞控

**实操任务**：
1. ESP32 通过 MQTT 每隔 5 秒上报一条模拟数据到服务器：
   ```json
   {"device": "drone-01", "battery": 15.2, "gps_lat": 39.12, "gps_lon": 117.35, "alt": 120}
   ```
2. 服务器通过 MQTT 下发指令，ESP32 收到后点亮/熄灭 LED（模拟飞机收到指令）
3. ESP32 串口连接飞控 TELEM1，实现双向透传：
   - ESP32 串口收到 MAVLink 数据 → MQTT 发到服务器
   - 服务器 MQTT 下发的指令 → ESP32 串口发给飞控

**经验预警**：
- 🌐 MQTT broker 推荐用 EMQX 公有云（免费额度够你用），或自搭 Mosquitto
- 🌐 MQTT 消息不要太大！保持在 1KB 以内，4G 网络大包容易丢
- 🌐 QGC 走 UDP 连接代理要配好 MAVLink 心跳，否则 5 秒断连

---

### 阶段三：研究 4G/5G 通信模块（3-4 周）

**目标**：EC20 模块拨号上网，实现 4G 环境下的数据通信。

> 注意：你们买的 EC20 是 4G 模块，"5G"可能是项目申报书里写的。现阶段先用 4G 验证，5G 模块（如 RM500U）可以后期升级。

**学习内容**：
- 4G 模块的工作模式：AT 命令模式 vs PPP/QMI 拨号模式
- EC20 的 USB 模式切换（AT 命令 `AT+QCFG="usbnet"`）
- PPP 拨号脚本（Linux 下 `pppd`）/ QMI 协议拨号（`libqmi`）
- 4G 公网 IP vs 内网 IP —— 两个 EC20 如何互联是大问题

**实操任务**：
1. EC20 USB 插电脑，用串口助手发 `AT`，收到 `OK`（确认模块活着）
2. 发 `AT+CSQ` 查看信号强度
3. 发 `AT+COPS?` 查看注册的运营商
4. EC20 插到树莓派，配置 PPP 拨号，拿到 IP 地址
5. 树莓派通过 4G ping 通外网
6. 两个 EC20 互通测试（可能需要内网穿透，先准备 frp/n2n 方案）

**经验预警**：
- 📶 4G 模块必须在有天线的情况下上电！不接天线发射可能烧功放
- 📶 EC20 的 USB 转接板可能需要额外供电，USB 口直接插电脑有时供电不足
- 📶 **4G 内网穿透是最大的坑**：国内运营商给 SIM 卡分配的是内网 IP（NAT），两个 4G 设备不能直接互联。解决方案：
  - 方案 A：买一张带公网 IP 的物联网卡（贵，不一定有）
  - 方案 B：用云服务器做中转（¥50/月 的轻量云服务器就够了）
  - 方案 C：frp/n2n/ZeroTier 内网穿透（免费，延迟高一点）

---

### 阶段四：中台系统设计（持续迭代）

**目标**：搭建服务器，把换电站、飞机、地面站连成一个系统。

**学习内容**：
- 中台架构：MQTT Broker + 数据库 + Web API + 前端展示
- 数据流设计：飞机上报 GPS/电池/状态 → 中台存储 → 地面站展示
- 指令下发：地面站 → 中台 → 换电站/飞机
- 数据库选型：InfluxDB（时序数据，适合飞行日志）+ SQLite/MySQL（业务数据）

**实操任务**：
1. 在云服务器上用 Docker 搭 Mosquitto MQTT Broker
2. 写一个 Python 脚本订阅所有 MQTT 消息并存入数据库
3. 做一个最简单的 Web 页面，实时显示飞机位置（用 Leaflet.js 地图库）
4. 实现"飞控低电量 → MQTT 上报 → 中台检测 → 自动下发 RTL 指令"

**推荐技术栈**（学习成本低、资料多）：
- 后端：Python + FastAPI / Flask
- 数据库：SQLite（前期）+ InfluxDB（飞行数据）
- 前端：Vue.js / 或者直接 QGC（省掉前端开发）
- MQTT Broker：EMQX / Mosquitto
- 服务器：阿里云/腾讯云轻量应用服务器（学生价 ~¥50/月）

**经验预警**：
- 🖥 别在本地电脑跑服务器！要么买云，要么用一台旧电脑装 Linux。本地跑的话，飞机不可能连到你的电脑
- 🖥 前期不需要做漂亮的前端界面——QGC 可以直连 MQTT 桥接后的 MAVLink，先用 QGC 看数据
- 🖥 安全！MQTT Broker 必须加用户名密码，别裸奔在公网上

## 4. 3D 建模与结构设计（秋森 & 用户）

### 整体路径

```
精进建模 → 为其他部分设计模型 → 3D 打印迭代
```

### 阶段一：精进建模技能（持续学习）

**工具选择**：你们已经有 SolidWorks（`样板.SLDPRT`），继续用 SolidWorks。

**学习内容**：
- 参数化建模 —— 不要"画到哪里算哪里"，尺寸用变量驱动，方便后期改
- 装配体功能 —— 把多个零件组装在一起，检查干涉
- 设计用于 3D 打印的设计规则：
  - 悬空角度 < 45°（否则要加支撑）
  - 孔直径比实际大 0.2-0.3mm（打印收缩）
  - 壁厚 ≥ 1.2mm
  - 螺纹不要直接打印（打印后攻丝或嵌入金属螺纹套）
- 结构受力分析（SolidWorks Simulation，基础版的够用了）

**实操任务**：
1. 打开 `样板.SLDPRT`，对照实物理解建模思路
2. 把现有的 DM-FC01 飞控测量一遍，建一个精确的 3D 外壳模型，打印出来试装
3. 设计一个简单的舵机支架（用于投放舱），打印验证

**经验预警**：
- 🖨 **3D 打印的第一条规则：你设计的东西不一定能打印出来**。和打印服务商或机器确认材料（PLA/PETG/ABS）
- 🖨 PLA 在阳光下暴晒会软化变形，飞行器结构件尽量用 PETG 或 ABS
- 🖨 打印精度在 ±0.2mm 左右，配合面要留间隙

---

### 阶段二：为其他组设计模型（贯穿整个项目）

**待设计清单**：

| 零件 | 对接小组 | 关键约束 |
|------|----------|----------|
| DM-FC01 飞控外壳 | 飞机组 | 减震安装，通风散热 |
| 树莓派 + EC20 外壳 | 通信组 | USB 线走线，天线 SMA 开孔 |
| 电池仓 | 电控组 | 弹簧触点位置，快拆结构 |
| 换电站对接平台 | 电控组 | ArUco 码安装位置，电气触点位置 |
| 投放舱 | 电控组 | 舱门铰链，舵机安装位，重力方向 |
| 降落伞舱 | 电控组 | 触发机构空间，弹射方向 |
| 电机座 | 飞机组 | 电机螺丝孔位，散热 |
| GPS / 天线支架 | 飞机组 | 远离干扰源，减震 |
| 4S 电池绑带/固定座 | 全部 | 重心位置，防脱落 |

**设计流程**：
1. 和对应小组沟通需求（尺寸、功能、受力方向）
2. 手绘草图确认
3. SolidWorks 建模
4. 导出 STL → 3D 打印
5. 实物测试 → 发现问题 → 改模型 → 重新打印

**经验预警**：
- 📐 设计前先拿到实物！根据网上的尺寸建模 99% 会翻车
- 📐 每个零件预留 2-3 次迭代的打印时间。第一次打印出来几乎一定有问题
- 📐 和电控组保持沟通——他们需要的安装孔距、走线空间你可能没考虑到

## 5. 通用踩坑指南

### 所有组通用

| 坑 | 说明 | 教训 |
|----|------|------|
| **驱动地狱** | Windows 上 ST-Link/USB-TTL/CP210x 驱动经常冲突 | 准备一个 Linux 的 U 盘 Live 系统做调试，省无数时间 |
| **供电是万恶之源** | 80% 的"代码问题"其实是供电不足或不稳 | 万用表在手边，怀疑代码前先量电压 |
| **接地** | 不同模块共地才能通信，这是电工常识但初学者不知道 | 所有模块的 GND 连在一起 |
| **先买后设计** | 先画图再买件 → 买回来的件尺寸不对 | 先买件，量实物，再画图 |
| **只看中文资料** | 无人机领域的精华资料大都是英文的 | 至少要能读英文 datasheet 和 GitHub README |
| **一个人闷头搞** | 出问题不跟队友说，自己 debug 两天 | 群里说一句，可能别人刚好遇到过 |

### 飞控专属

| 坑 | 说明 |
|----|------|
| USB 不供电 | DM-FC01 的 5V/8.9V 输出必须接电池才有，USB 只给 MCU 供电 |
| SBUS 反相器 | DM-FC01 的 SBUS 需要 PE15 拉高才使能，别忘了 |
| IMU 旋转方向 | 飞控安装方向错了 → 起飞瞬间翻。默认 ROTATION_PITCH_180 |
| 电池电压校准 | BATT_VOLT_MULT 默认值 10.2，务必用万用表校准 |

### 4G 通信专属

| 坑 | 说明 |
|----|------|
| 天线没接就上电 | 烧 4G 模块功放，不可逆 |
| 内网 IP | 两个 4G 设备不能直连，必须有中转服务器 |
| 延迟 vs 带宽 | 4G 上传带宽可能只有 5-10Mbps，视频码率要控制在 2-4Mbps |
